# Tag for Everything

Мощный плагин для Obsidian для массового управления тегами. Быстро добавляйте теги ко всем заметкам в папке, находите заметки без тегов и автоматически организуйте их.

## Возможности

### 1. Массовое добавление тегов в папку
- Выберите любую папку в вашем хранилище
- Добавьте несколько тегов одновременно ко всем существующим заметкам в этой папке
- Теги добавляются в YAML frontmatter
- Сохраняет существующие теги (без дубликатов)
- Работает с вложенными папками

### 2. Управление заметками без тегов
- Автоматически обнаруживает все заметки без тегов
- Показывает количество заметок без тегов
- Добавляет определенный тег ко всем заметкам без тегов одним нажатием
- Автоматически перемещает заметки без тегов в назначенную папку
- Помогает поддерживать порядок в хранилище

## Использование

### Добавление тегов в папку

1. Откройте Настройки → Tag for Everything
2. В разделе "Add tags to existing notes in folder":
   - Введите путь к папке (например, `Проекты/Работа` или просто `Работа`)
   - Введите теги через запятую (например, `проект, важное, работа`)
   - Нажмите "Apply Tags to Folder"

Все существующие заметки в этой папке теперь будут иметь эти теги в frontmatter.

### Управление заметками без тегов

1. Откройте Настройки → Tag for Everything
2. В разделе "Untagged notes management":
   - Нажмите "Refresh", чтобы увидеть количество заметок без тегов
   - Укажите папку, куда следует переместить заметки без тегов (по умолчанию: `Untagged Notes`)
   - Укажите тег для добавления к заметкам без тегов (по умолчанию: `untagged`)
   - Нажмите "Process Untagged Notes"

Все заметки без тегов будут помечены тегами и организованы в указанной папке.

## Установка

### Из Obsidian Community Plugins (Рекомендуется)

1. Откройте Настройки в Obsidian
2. Перейдите в Community Plugins и отключите Safe Mode
3. Нажмите Browse и найдите "Tag for Everything"
4. Нажмите Install, затем Enable

### Ручная установка

1. Скачайте последний релиз с GitHub
2. Извлеките файлы в папку `.obsidian/plugins/tag-for-everything/` вашего хранилища
3. Перезагрузите Obsidian
4. Включите плагин в Настройки → Community Plugins

## Как это работает

### Добавление тегов
Плагин добавляет теги в YAML frontmatter ваших заметок. Если у заметки нет frontmatter, он создаст его. Например:

```markdown
---
tags: [проект, работа, важное]
---

# Содержание заметки
```

### Обнаружение тегов
Заметка считается "без тегов", если у нее:
- Нет YAML frontmatter с полем `tags:`
- Нет встроенных тегов (например, #тег) в содержимом

## Настройки

| Настройка | Описание | По умолчанию |
|-----------|----------|--------------|
| Target folder | Папка для добавления тегов | (пусто) |
| Tags to add | Список тегов через запятую | (пусто) |
| Folder for untagged notes | Куда переместить заметки без тегов | `Untagged Notes` |
| Tag for untagged notes | Тег для добавления к заметкам без тегов | `untagged` |

## Примеры

### Пример 1: Добавление тегов в папку проекта
У вас есть папка `Проекты/РедизайнСайта` с 50 заметками. Вы хотите пометить их всех тегами `проект`, `сайт` и `2024`.

1. Укажите Target folder: `Проекты/РедизайнСайта`
2. Укажите Tags to add: `проект, сайт, 2024`
3. Нажмите "Apply Tags to Folder"

Все 50 заметок теперь имеют эти три тега.

### Пример 2: Организация заметок без тегов
У вас есть 100 заметок, разбросанных по хранилищу без каких-либо тегов.

1. Нажмите "Refresh", чтобы увидеть количество
2. Укажите Tag for untagged notes: `требует-проверки`
3. Укажите Folder for untagged notes: `На проверку`
4. Нажмите "Process Untagged Notes"

Все 100 заметок теперь помечены тегом `требует-проверки` и перемещены в папку `На проверку`.
